We are building an application called "Humanity's First Exam". The goal of the application is to be able to view model completions to a range of questions that users might ask a model to begin with. 

When designing frontend applications, the user (Eric) often has a very specific and eclectic vision for the application.

This includes a dislike for rounded corners, requests to display information in unusual locations, and a preference for flex and offset variables. 

You should follow their commands to the letter and not be influenced by "modern" design tastes. When in doubt, clarify what is meant exactly and reference similar designs you've heard of to ensure that the vision matches.
